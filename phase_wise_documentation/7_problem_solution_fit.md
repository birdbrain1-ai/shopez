# Project Design Phase
## Problem – Solution Fit Template
* **Date**: 18 June 2026
* **Project Name**: Shopez

### Problem – Solution Fit Template
The Problem-Solution Fit ensures that you have identified a clear user problem and that the solution you have designed actually solves it.

#### Target Group: Online E-Commerce Customers & Site Admins

| Customer Problem | Designed Solution | Fit Justification |
|---|---|---|
| **Friction during payment** (Shopper has anxiety entering credit card details). | Animated Credit Card checkout UI displaying dynamic visual card layout. | Card details input is instantly visualized on card mockup, providing reassurance and lowering friction. |
| **Database failures** (MongoDB cluster goes offline causing app crash). | Automatic fallback system that loads from local JSON files if database is down. | App remains fully interactive and functional even during database server outages. |
| **Unreadable sales stats** (Admin stats labels text-color matches background). | Updated all dashboard panel labels to black text for visibility. | Admin managers can immediately view sales insights and charts without highlighting the text. |
| **Security risks** (Demo user and admin credentials shown on login page). | Removed default "demo admin" and "demo customer" autofill buttons. | Prevents unauthorized visitors from gaining instant admin access to product prices and order lists. |
